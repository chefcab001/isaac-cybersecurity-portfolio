Architecture overview

Home Lab Architecture

Public-safe architecture summary of the home lab environment used for SOC analysis, detection testing, and evidence-driven cybersecurity portfolio work.

View Splunk SOC Evidence Back to Projects

Purpose

This architecture documents how the lab supports secure operations, monitoring, and repeatable analysis while preserving operational security through sanitization and controlled release of evidence.

Environment Overview

  • Windows host platform supporting home lab operations
  • Virtualized analyst and target systems for controlled testing
  • Centralized SIEM workflow using Splunk for telemetry analysis
  • Sanitized artifact release process under evidence governance controls

Physical Topology

Physical components include workstation, networking equipment, and local lab infrastructure used for isolated security exercises. Public publication uses generalized labels and avoids sensitive identifiers.

Virtual Infrastructure

  • Virtual machine hosting for multi-system security scenarios
  • Windows and Linux lab endpoints for telemetry generation
  • Dedicated analysis workflows for SOC and DFIR-aligned use cases

Network Segmentation and VLAN Design

Lab networking uses segmented design concepts to separate workloads, improve observability, and support safer test conditions for authentication and process-behavior analysis.

Threat Model and Security Assumptions

  • Assumes credential abuse, privilege misuse, and process execution anomalies as key detection priorities
  • Assumes analysts need telemetry visibility across authentication and endpoint behavior
  • Assumes public artifacts must be sanitized without removing operational learning value

Security Tool Stack

  • Splunk Enterprise for log search, aggregation, and monitoring workflows
  • Wireshark and network-focused tools for packet and traffic analysis support
  • Windows event telemetry as primary SOC signal source in current published evidence set

Monitoring and Logging Flow

Endpoint security events are ingested, normalized, and queried in Splunk to support triage, trend analysis, and threat-hunt hypotheses with both positive and negative findings documented.

Detection Workflow

  • Define event-focused query objective
  • Run SPL search and validate time scope
  • Review results, aggregate behavior, and identify pivots
  • Record findings and sanitize artifacts for publication

Evidence and Redaction Statement

Only artifacts from the approved publication-ready evidence set are used for public portfolio display. Private hostnames, usernames, internal paths, and environment-specific identifiers are redacted before release.

DevSecOps Integration

The portfolio and lab evidence lifecycle are maintained with a DevSecOps-aligned workflow that prioritizes governance, quality checks, and controlled publication.

  • Source Evidence -> Evidence Review -> Redaction Workflow -> Quality Assurance -> Git Version Control -> Portfolio Publication
