# Portfolio Workflow

When working on this repository:

Analyze the existing codebase before making changes.

Explain:

- What currently exists.
- What improvements are recommended.
- Which files will be modified.

Implement changes incrementally.

After each change provide:

- Files changed
- Summary of modifications
- Validation performed
- Remaining tasks

Never rebuild the site from scratch unless explicitly instructed.

Prioritize:

- Professional appearance
- Mobile responsiveness
- Accessibility
- Recruiter experience
- Performance
- Cybersecurity branding

Always suggest portfolio evidence screenshots after significant changes.
