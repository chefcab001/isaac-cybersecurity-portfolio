Executive Summary

This lab demonstrates SOC analyst workflow using sanitized evidence from a working Splunk environment. All publication artifacts were moved through evidence governance, redaction review, and public-release controls before being included here.

Skills Demonstrated

Environment Overview

Splunk Enterprise was used to query Windows Security Event telemetry with focus on authentication, privilege, credential usage, process creation, and event-volume trend analysis.

  • Primary source: WinEventLog:Security
  • Supporting source validation: System/Application/Setup sourcetypes
  • Analysis method: targeted SPL searches plus aggregate views

Methodology

  1. Validate source ingestion and sourcetype readiness.
  2. Run event-focused SPL hunts across authentication and process events.
  3. Pivot and correlate findings across related Event IDs.
  4. Document positive and no-result outcomes for transparent reporting.
  5. Route artifacts through redaction workflow before publication.

Log Ingestion Validation

Ingestion and parsing validation confirmed searchable Windows event telemetry and usable sourcetype categorization for SOC triage and hunt workflows.

Detection Engineering

  • Event ID 4624: successful authentication analysis
  • Event ID 4625: failed logon hunt with documented negative findings
  • Event ID 4634: logoff event activity review
  • Event ID 4648: explicit credential usage monitoring
  • Event ID 4672: privileged logon event review
  • Event ID 4688: process creation and top process frequency analysis
  • Event ID 4768: Kerberos authentication hunt with documented negative findings

SPL Query Examples

  • source="WinEventLog:Security" EventCode=4624 | top Account_Name
  • source="WinEventLog:Security" EventCode=4625 | top Account_Name
  • source="WinEventLog:Security" | top limit=10 EventCode
  • source="WinEventLog:Security" EventCode=4688 | top New_Process_Name
  • source="WinEventLog:Security" EventCode=4648
  • source="WinEventLog:Security" | timechart count

Findings or Outcomes

  • Authentication and logoff telemetry supports account activity monitoring workflows.
  • Privileged and explicit credential events provide high-value investigation pivot points.
  • Process creation visibility enables baseline behavior review and suspicious execution triage.
  • No-result hunts were documented as valid outcomes to support baseline confidence.

Lessons Learned

Not all searches returned matching events in the selected time range. These negative outcomes were preserved as valid evidence of disciplined SOC analysis and transparent reporting.
