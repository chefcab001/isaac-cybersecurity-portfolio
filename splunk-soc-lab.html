Splunk SOC Analyst Home Lab

A public-safe lab project focused on Windows security telemetry, Sysmon, Splunk Enterprise, detection searches, MITRE ATT&CK mapping, alert creation, and incident investigation workflow.

Back to Projects

Project Overview

This home lab is designed to demonstrate entry-level SOC analyst readiness through controlled endpoint telemetry collection, SIEM ingestion, detection development, alert review, and investigation notes. All documentation is sanitized and avoids private IPs, usernames, hostnames, credentials, and sensitive data.

Lab Architecture

The planned architecture uses a Windows endpoint, Sysmon telemetry, and Splunk Enterprise running locally or in a virtual machine. Evidence will use generic asset names and sanitized diagrams rather than exposing real system identifiers.

Core Build

  • Windows endpoint
  • Sysmon installed and configured
  • Splunk Enterprise installed locally or in a VM
  • Windows Event Logs ingested into Splunk
  • Sysmon logs ingested into Splunk
  • Basic dashboard for security events

Detection Work

  • Failed logon detection
  • Suspicious PowerShell activity
  • Process creation monitoring
  • New local admin detection
  • Possible credential access indicators
  • MITRE ATT&CK mapping for each detection

MITRE ATT&CK Mapping

Detection notes will map each search to relevant ATT&CK tactics and techniques, such as credential access, execution, persistence, discovery, and defense evasion where supported by lab telemetry.

Portfolio Evidence Plan

  • Network/lab diagram
  • Screenshots of Splunk ingestion
  • Screenshots of SPL searches
  • Alert rule examples
  • Incident investigation walkthrough
  • Lessons learned
  • Sanitized findings and analyst notes

Analyst Workflow

The investigation workflow will document alert triage, event review, timeline building, hypothesis testing, MITRE mapping, analyst notes, and recommendation writing in a repeatable SOC-style format.

Resume Bullet

Built a Splunk SOC Analyst home lab using Windows, Sysmon, and Splunk Enterprise to ingest security telemetry, create detection searches, map activity to MITRE ATT&CK, and document incident investigation workflows.

Lessons Learned

The project will focus on practical lessons from data onboarding, source validation, false-positive review, detection tuning, alert context, and communicating investigation results clearly.
